Executive Summary

A Medallion architecture (Bronze / Silver / Gold) on S3 with Apache Iceberg, powered by Kinesis + Glue Streaming for real-time ingestion, AWS Glue for batch ETL, Amazon Redshift for warehousing, and QuickSight for BI. Designed for daily growth from 1 GB to 20+ GB/day with cost-optimised auto-scaling, zero single points of failure, and full governance via Lake Formation + KMS.

Key Design Decisions

Why Kinesis over MSK at launch

At 1 GB/day, Kinesis costs ~$50/month vs MSK's $400+/month minimum. Use Kafka-compatible producer API from day 1 to enable zero-code migration to MSK when throughput exceeds 10 GB/day or multi-team consumption is needed.

Why Iceberg over Hudi/Delta

Iceberg is AWS's preferred table format with native support in Athena, Glue, EMR, and Redshift Spectrum. Hidden partitioning, schema evolution without rewrites, time travel for debugging, and ACID transactions. Best ecosystem fit for this AWS-native stack.

Why Glue over EMR for ETL

Serverless Glue eliminates cluster management. Job bookmarks enable incremental processing out-of-the-box. Integrated with Glue Data Catalog. When data grows beyond ~100 GB/job, migrate to EMR Spot for 60-80% cost savings.

Why MWAA over Step Functions

For complex multi-step pipelines with 10+ tasks, cross-system dependencies, and team familiarity with Airflow DAGs, MWAA provides richer orchestration. Use Step Functions for AWS-native sub-workflows embedded within Airflow DAGs.

Recommended Production Architecture

Detailed Component Breakdown

Layer 1: Data Ingestion Ingestion

Real-time events → Kinesis: 5 shards initially (5 MB/s capacity). Enhanced fan-out. 7-day retention for replay.

CDC → DMS: Ongoing replication from RDS/Aurora. Deliver to Kinesis or directly to S3 Bronze. t3.medium for <1 GB/day.

File drops → S3 + Lambda: S3 event notification → Lambda validates/routes → Step Functions orchestrate.

REST APIs → API Gateway → Kinesis: Direct integration (no Lambda). Reduces cost and latency.

Layer 2: Stream Processing Streaming

Kinesis → Firehose: Buffer 300s or 128 MB. Transform via Lambda (JSON→Parquet). Deliver to S3 Bronze partitioned by date.

Kinesis → Glue Streaming: When real-time enrichment needed (join with DynamoDB dim tables). 5-min micro-batch. Writes to S3 Silver directly.

DLQ: Malformed records → S3 DLQ prefix. Lambda alert via SNS. Manual review + Kinesis replay.

Layer 3: Medallion Data Lake Storage

Bronze: s3://bucket/bronze/ · Partition: source/year/month/day/hour · Parquet · Versioning ON · Lifecycle: Standard 90d → IA → Glacier 1yr.

Silver: s3://bucket/silver/ · Iceberg table format · Partition: source/date · Cleaned, validated, deduped · Glue Catalog · LF column security on PII.

Gold: s3://bucket/gold/ · Aggregated by business domain · Partition: domain/date · Star schema · Optimized for Redshift Spectrum + BI.

Layer 4: ETL/ELT Processing Processing

Bronze→Silver Glue Job: Every 30 min. Job bookmarks incremental. Validate schema, dedup on (entity_id, event_time), mask PII, write Iceberg Silver.

Silver→Gold Glue Job: Daily at 2AM. Join fact + dimension tables. Compute KPIs. Write Gold Parquet + COPY to Redshift.

Compaction Job: Weekly Glue Iceberg OPTIMIZE. Merges small files into 256 MB+ Parquet. Reduces Athena scan cost by 30%.

Layer 5: Redshift Warehouse Warehouse

Config: RA3.xlplus × 2 nodes. Separate storage (S3-backed). Scheduled pause 10PM-6AM (saves 67% off-hours).

Spectrum: External tables → S3 Gold Iceberg. Query cold data without loading. Hot data (last 30 days) in native Redshift tables.

Optimization: Sort keys on date + entity_id. DISTKEY on high-join columns. Materialized views for dashboards. WLM queues for BI vs ETL.

Layer 6: Security & Governance Security

Encryption: S3 SSE-KMS with CMK. Redshift + Kinesis encrypted. KMS key rotation annually. All data in transit: TLS 1.2+.

Access: Lake Formation column-level security on Silver PII (mask email, phone for non-admin). Row-level filters for multi-tenant. IAM least-privilege roles per service.

Audit: CloudTrail all S3/Glue/Redshift API calls. LF audit logs. Redshift audit logging to S3. Weekly IAM access review.

Cost Optimization Techniques

S3 Storage Optimization

S3 Intelligent-Tiering: Enable on Bronze layer. AWS auto-moves objects between tiers. Saves 30-40% on rarely accessed data. No retrieval fees for Frequent/Infrequent tiers.

Lifecycle Policies: Bronze: Standard 90d → IA 90d → Glacier 1yr. Silver: Standard 180d → IA. Gold: Standard always. Saves $0.019/GB vs Standard.

Columnar Compression: Parquet + Snappy reduces storage 60-75% vs raw JSON. Reduces Athena scan cost proportionally.

Compute Cost Optimization

Redshift Scheduled Pause: Pause 10PM–6AM + weekends. Saves 67% of compute cost. RA3 storage persists during pause.

EMR Spot Instances: r5.xlarge Spot ~$0.06/hr vs on-demand $0.25/hr. 60-80% savings. Use instance fleets for Spot availability.

Kinesis On-Demand: Auto-scales shards during rapid growth phase. Eliminates hot-shard throttling during spikes.

Query Cost Optimization

Athena Partitioning: Always filter on partition columns. Reduces cost from $500/month to $50/month on same data volume.

Materialized Views: Pre-compute expensive aggregations. Dashboard queries hit MV (fast) instead of raw tables (expensive).

Result Caching: Athena 24h cache. Redshift result cache. QuickSight SPICE eliminates per-refresh Redshift compute cost.

Advanced Data Engineering Topics

Bronze (Raw): Exact copy of source data. No transformation. Schema-on-read. S3 path: s3://bucket/bronze/source=salesforce/year=2024/month=01/day=15/. Format: Raw JSON or Parquet. Retention: 2 years in Standard, then Glacier. Purpose: audit trail, reprocessing source for Silver.
Silver (Curated): Cleaned, validated, deduplicated. Schema-on-write with Iceberg. Business rules applied. PII masked (SHA-256 hash or tokenized). Null handling. Duplicate detection on event_id + event_timestamp. Schema validation via Glue Schema Registry.
Gold (Business): Pre-aggregated, domain-specific tables. Examples: daily_sales_by_region, hourly_active_users, monthly_churn_rate. Optimized for BI tools. May be in Redshift (fast queries) or S3 + Redshift Spectrum. Partitioned by business date dimensions.
At-Least-Once: Messages may be delivered multiple times. Simpler to implement. Requires idempotent downstream processing. Kinesis KCL uses checkpoint in DynamoDB — on restart, reprocesses from last checkpoint. May process last batch twice. Mitigation: MERGE/UPSERT instead of INSERT; unique dedup key in target.
Exactly-Once: Each message processed exactly once. Requires distributed transactions or idempotent producers + transactional consumers. Kafka transactions (beginTransaction/commitTransaction) provide exactly-once between Kafka topics. Kinesis → Lambda → DynamoDB can achieve exactly-once with conditional writes.
Practical Advice: At-least-once + idempotent writes = effectively-once. In practice, full exactly-once semantics have significant overhead (2x latency). Design for at-least-once with MERGE semantics in Gold layer. This is the pragmatic production standard.

Glue Job Bookmarks: Track the last processed S3 key/DynamoDB record. On next run, start from where it left off. Simple, built-in, works for append-only workloads.
Watermark-based: Track max(event_timestamp) per source in DynamoDB. Next run queries: WHERE event_timestamp > last_watermark. Handles late-arriving data with configurable tolerance window (e.g., accept data up to 24 hours late).
Partition-based: Process only new S3 partitions (year=2024/month=01/day=15). Glue job scans Glue Catalog for partitions added after last_run_time. Most efficient for daily batch pipelines.
Iceberg Incremental Scan: Use Iceberg's incremental append — read only files added between snapshot_id_1 and snapshot_id_2. No watermarks needed. Works with late data and schema evolution simultaneously.
Root Causes: High-frequency Firehose writes (every 60s → 1,440 files/day), high partition cardinality (date × region × product = thousands of partitions each with 1 file), streaming micro-batch every 5 minutes.
Prevention: Firehose buffer window 300-900s for >100 MB files. Spark coalesce(numPartitions) before writing. Iceberg hidden partitioning merges small writes into larger files automatically.
Reactive Compaction: Daily Glue/EMR Spark job: spark.read.parquet(partition_path).coalesce(1).write.parquet(partition_path). Iceberg OPTIMIZE TABLE (rewrites small files ≥128 MB). VACUUM to remove orphan files.
Target: 128 MB – 1 GB per Parquet file for optimal Athena/Spark read performance. Track file count per S3 prefix in CloudWatch. Alert when average file size < 50 MB.
CDC Options: (1) AWS DMS with ongoing replication — captures WAL logs from Postgres/MySQL, delivers to S3 or Kinesis. (2) Debezium on MSK Connect — open-source CDC connector, more flexible, better community support. (3) Aurora MySQL has native binlog streaming. Choose DMS for simplicity, Debezium for Kafka-native architectures.
Schema Evolution: (1) Use Apache Iceberg/Hudi — both support schema evolution (add columns, rename fields) with backward/forward compatibility. (2) AWS Glue Schema Registry with Avro/JSON Schema — validates messages at ingestion. (3) Glue Catalog schema versioning — tracks all historical schemas. Strategy: allow additive changes automatically; breaking changes require a new table version or migration script.
Delta vs Iceberg vs Hudi (AWS):
• Iceberg — AWS preferred format. Native: Athena, Glue, EMR, Redshift Spectrum. Best schema evolution + hidden partitioning. RECOMMENDED for new AWS projects.
• Hudi — Best for CDC/upsert-heavy workloads. MoR vs CoW write modes. Strong DMS integration. Used in fintech for real-time CDC.
• Delta Lake — Best for Databricks users. Native on EMR but less AWS-native integration. Excellent time travel.

Final Recommendation

Start with: Kinesis Data Streams (5 shards) + Firehose → S3 Bronze (Parquet) → Glue ETL (Iceberg Silver/Gold) → Redshift RA3.xlplus (2 nodes, scheduled pause) → QuickSight → MWAA orchestration. Total: ~$990/month. Scale Kinesis shards as throughput grows. Migrate to MSK when multi-consumer or >10 GB/day. Adopt EMR Spot for large Spark jobs at 20 GB/day for 60-80% ETL cost savings. Use Apache Iceberg table format throughout Silver/Gold for schema evolution, ACID transactions, and Athena/Glue/Redshift compatibility.
